Cloudflare Pages — settings to use with this repo
==================================================

In the Pages project: Settings → Builds & deployments → Build configuration

  Framework preset:        None
  Build command:           bash build.sh
  Build output directory:  .
  Root directory:          (leave empty = repo root)

Deploy command:           (leave COMPLETELY EMPTY)
  Git-connected Pages publishes the build output automatically after build.sh
  exits 0. You do not replace that with Wrangler.

  In the Deploy command field, do NOT use any of these:
    npx wrangler deploy
    npx wrangler pages deploy ...
  - wrangler deploy targets a Worker (needs main / assets); Wrangler will warn
    it is wrong on a Pages project and fail with "Missing entry-point".
  - wrangler pages deploy is only for Direct Upload pipelines, needs a valid
    CLOUDFLARE_API_TOKEN with Pages permissions, and is redundant with Git Pages.

Production env vars (Settings → Environment variables):

  Do NOT add CLOUDFLARE_API_TOKEN for normal Git Pages builds — it is not used
  unless you intentionally run wrangler from the build, and wrong tokens cause
  auth error 10000 on pages deploy.

  GITHUB_UPLOAD_TOKEN   (required) — GitHub PAT for commits + PRs
  GITHUB_OWNER          (optional) default y1-community
  GITHUB_REPO           (optional) default InnioasisY1Themes
  GITHUB_BASE_BRANCH    (optional) default main
  GITHUB_ZIP_UPLOAD_DIR (optional) prefix path for uploaded zips
  MAX_UPLOAD_BYTES      (optional) default 26214400 (25 MiB)

If your Pages project name is not "y1themes", edit the `name` field in wrangler.toml
to match the dashboard (or rename the project to match the file).

Build image: use V2 or later if prompted (needed for wrangler.toml Pages config).

Troubleshooting: "Upload failed (404)"
--------------------------------------
The upload form POSTs to /api/upload-theme. That route is provided by **Pages
Functions** (./functions/), which run only on a **Cloudflare Pages** deployment.

If you open the site on ***.workers.dev**, you likely deployed a **Worker** with
static assets only — there is no Pages Function there, so /api/upload-theme
returns 404.

Fix: open upload.html on your **Pages** hostname (e.g. https://<project>.pages.dev/upload.html)
or attach your custom domain to the **Pages** project (not only to a Worker).

Advanced: host static HTML elsewhere but point uploads at Pages by setting the
meta tag in upload.html:
  <meta name="cf-pages-upload-api-origin" content="https://YOUR.pages.dev" />

Troubleshooting: wrangler pages deploy + "Authentication error [code: 10000]"
-----------------------------------------------------------------------------
You set a custom **Deploy command** like `npx wrangler pages deploy ...`. Remove
it entirely. For Git-connected Pages, deployment is automatic after `build.sh`.

If you removed the deploy command but still have CLOUDFLARE_API_TOKEN in env,
you can delete that variable unless another build step needs it.

Troubleshooting: "Missing entry-point to Worker script or to assets directory"
------------------------------------------------------------------------------
The **Deploy command** is set to `npx wrangler deploy`. Remove it (empty field).
This repo is a Pages site + Pages Functions, not a Worker with `main = "..."`.
