Cloudflare Pages — settings to use with this repo
==================================================

In the Pages project: Settings → Builds & deployments → Build configuration

  Framework preset:        None
  Build command:           bash build.sh
  Build output directory:  .
  Root directory:          (leave empty = repo root)

Deploy command:           (leave COMPLETELY EMPTY)
  Git-connected Pages publishes the build output automatically after build.sh
  exits 0.

  Do NOT put any of these in the Deploy command field:
    npx wrangler deploy
    npx wrangler pages deploy ...
  See troubleshooting sections below.

Production env vars (Settings → Environment variables):

  Do NOT add CLOUDFLARE_API_TOKEN for normal Git Pages builds unless you
  intentionally run wrangler pages deploy from that field.

  GITHUB_UPLOAD_TOKEN   (required) — GitHub PAT for commits + PRs
  GITHUB_OWNER          (optional) default y1-community
  GITHUB_REPO           (optional) default InnioasisY1Themes
  GITHUB_BASE_BRANCH    (optional) default main
  GITHUB_ZIP_UPLOAD_DIR (optional) prefix path for uploaded zips
  MAX_UPLOAD_BYTES      (optional) default 26214400 (25 MiB)

If your Pages project name is not "y1themes", edit the `name` field in wrangler.toml
to match the dashboard (or rename the project to match the file).

Build image: use V2 or later if prompted (needed for wrangler.toml Pages config).

Troubleshooting: log still says "Executing user deploy command: npx wrangler deploy"
--------------------------------------------------------------------------------------
That line is ONLY configured in the Cloudflare dashboard — not in this repo.

  1) Workers & Pages → open the **Pages** project (not a Worker named y1themes).
  2) Settings → Builds & deployments → Build configuration.
  3) Find **Deploy command** (may be under "Advanced" or a second column / tab).
  4) **Delete the entire command** so the field is empty. Click **Save**.
  5) If you see separate **Production** and **Preview** build settings, clear
     Deploy command in **both**.
  6) Retry the deployment (Deployments → Retry, or push a commit).

If you cannot find an empty deploy field, search the project settings for
"wrangler" or "deploy" and remove any custom Worker deploy step.

Troubleshooting: "Missing entry-point to Worker script or to assets directory"
------------------------------------------------------------------------------
The Deploy command is `npx wrangler deploy`, which targets a **Worker**, not
Pages. This repo has no Worker `main` entry — remove the deploy command (above).

Troubleshooting: "Upload failed (404)" on upload.html
------------------------------------------------------
Use a **Pages** hostname (e.g. custom domain on the Pages project or *.pages.dev),
not a ***.workers.dev** Worker-only URL, unless you set cf-pages-upload-api-origin
in upload.html — see upload.html comments.
