ATTRIBUTION GUIDELINES
────────────────────────────────────────

When to attribute someone else's work:

If you build a theme completely from scratch, no credit to another theme creator is required.

If you start with a theme that you did not create and modify it in any way — including changes to images, layout, config files, asset sets, structure, spacing, or organization — attribution is required.

This is not about ownership of characters or source artwork. Attribution is about crediting the original theme package itself as the starting point — including the layout, configuration, curated assets, and the way the theme is assembled to function on the Innioasis Y1.

For example, the icons included in most themes are not simply downloaded from Google Images. Theme creators often edit, resize, crop, and adjust padding so the icons display correctly and align properly on the Y1 interface. The same applies to backgrounds, spacing, layout decisions, and how assets are mapped within the configuration files. When any of these elements are reused — whether partially or as a starting point — the work is considered derived from the original theme.

This means you may not take image files, icons, or other assets directly from another creator’s theme package and use them in your own theme without attribution.

Even if you only reuse a small number of files — such as a few icons — those assets still came from another theme and require credit. Taking files directly from another creator’s theme package is not the same as independently sourcing the original material and editing it yourself.

Just because an asset used in a theme may also exist elsewhere on the internet does not mean the processed file within a theme package can be reused without attribution. If you reuse the exact file included in someone else’s theme, credit is required.

If you do not wish to provide attribution, do not use any files that came from another theme package. Even partial reuse of theme content requires attribution.

Providing attribution does not mean the new theme is not your work. You may still identify the theme as created by you.

Attribution simply acknowledges the work that the new theme was built from.

Using another creator’s work and presenting it online as entirely your own violates license terms and undermines the standards of the theming community. If you created a fully original theme, you would expect credit if someone reused any part of it.

HOW TO CREDIT (EXAMPLES)
────────────────────────────────────────

Attribution can be written in many ways. The following are simple examples:

“Based on the ______ theme by ______”

“Edited from a theme by ______”

“Remix of ______ by ______”

“Icons reused from ______ by ______”

“Some icons reused from ______”

“Battery icons credit goes to ______”

If you are sharing modified files, it is also recommended to keep or update the author or description fields in the theme configuration file so it reflects that the theme was modified from another creator’s work.

For themes distributed under the mp3li license, attribution in the theme configuration description field and on any page where the theme is shared is required.

Attribution is standard practice across creative and developer communities. It allows people to trace the origin of work, recognize the effort of the original creator, and maintain a respectful ecosystem as the theming community grows.
